# DiTaaS
Diffusion Transformer as a Service

## 1) Описание `src/utils/dit_model.py`

`DiTModel` в текущем виде — это компактный baseline для работы с латентами `[B, C, H, W]` (или `[B, H, W, C]`) в формате query + condition:

- Латенты query и condition переводятся в токены (flatten по spatial, затем transpose).
- Для обеих веток используется линейная проекция в `hidden_size`.
- Временной шаг (`timestep`) кодируется синусоидальным эмбеддингом + MLP и добавляется к query-токенам.
- Основной блок `TransformerCrossAttentionBlock` состоит из:
  - self-attention по query,
  - cross-attention (query attends to condition),
  - MLP-блока,
  - pre-norm и residual-путей.
- После стека блоков: `LayerNorm` + линейная проекция обратно в латентные каналы.

Плюсы:

- Простая и стабильная структура, удобная для дебага.
- Быстро проверяет, учится ли модель общей геометрии/структуре сцены.

Ограничения, которые объясняют "формы есть, деталей не хватает":

- Токенизация без patch-механики и без адаптивной модуляции по времени ограничивает выразительность на сложных деталях.
- Нет явного механизма "сильного" conditioning внутри каждого подблока (только обычный cross-attn).
- При росте глубины (4 -> 16 блоков) улучшение может насыщаться, если узким местом становится не глубина, а способ conditioning/нормализации/целевой функции.

## 2) Описание `src/utils/dit_v2_model.py` и добавленных технологий

`DiTV2Model` уже содержит качественные апгрейды относительно `DiTModel`:

1. **Patchify через `Conv2d` (`query_patch_embedding`, `condition_patch_embedding`)**
   - Вместо "1 токен = 1 spatial cell" используется токенизация патчами.
   - Это повышает емкость представления на уровне локальных структур и сокращает длину последовательности (дешевле attention при большей глубине).

2. **2D sin-cos positional embedding**
   - Позиционка создается детерминированно из координат `(y, x)` и добавляется и в query, и в condition.
   - Обычно это стабильнее обучаемых positional table на длинных тренингах и лучше переносится на новые spatial-паттерны.

3. **AdaLN-Zero-подобная модуляция в `DiTV2Block`**
   - Из `timestep_states` генерируются `shift/scale/gate` для self-attn, cross-attn и MLP.
   - Инициализация modulation-слоя нулями (zero init) делает старт обучения более стабильным.
   - Gating позволяет постепенно "включать" вклад подблоков по мере обучения, что обычно помогает сходимости на глубокой модели.

4. **Финальная timestep-conditioned модуляция перед выходом**
   - После `final_norm` применяется отдельная `final_modulation(shift, scale)`, затем unpatchify в исходное spatial-поле.
   - Это усиливает контроль над выходом на разных уровнях шума.

Что это дало концептуально:

- Модель стала ближе к современному DiT-подходу (patch tokens + adaptive conditioning).
- Увеличился потенциал по деталям и устойчивости на длинном обучении.

Почему качество все еще может быть "похоже, но не дотягивает":

- Часто ограничивает уже не только архитектура, но и **loss design, sampling распределение timesteps, optimizer/schedule, EMA, data curriculum, и inference-процедура**.
- Для заметного рывка в деталях обычно нужен пакет изменений "архитектура + тренинг + инференс", а не один параметр типа количества блоков.

## 3) Технологии, которые стоит внедрить дальше

Ниже список приоритетных улучшений, сфокусированных на деталях, метриках и стабильной сходимости на большом числе итераций.

### 3.1. Модульная декомпозиция модели (высокий приоритет)

Разделить текущую модель на независимые структурные блоки:

- `InputAdapter` — приведение входов, patchify/tokenize, positional embeddings, optional input normalization.
- `TimeConditioner` — timestep embedding, возможно sigma/noise-level embedding, генерация modulation-параметров.
- `Backbone` — стек блоков внимания (можно легко менять тип: full attn, windowed, hybrid, U-ViT style).
- `OutputHead` — финальная нормализация, timestep-conditioned head, unpatchify/декодирование в нужный target.

Почему это важно:

- Резко упрощает расширение (новые conditioning-типы, разные backbones).
- Можно быстро сравнивать варианты, не переписывая всю модель.
- Проще делать ablation: где реально теряются детали (в input, backbone, или head).

### 3.2. Улучшение conditioning-механизма (высокий приоритет)

- Добавить **двунаправленный обмен** между query/condition (не только query <- condition, но и controlled condition updates).
- Ввести **multi-scale conditioning**: отдельные condition tokens для coarse/mid/fine уровней.
- Рассмотреть **cross-attention с learned key/value adapters** (или low-rank adapters) для более емкого переноса контекста.

### 3.3. Loss и objective для деталей (высокий приоритет)

- Перейти к более стабильной parameterization:
  - `v-prediction` или `epsilon+v` hybrid (вместо чистого epsilon, если сейчас он используется).
- Добавить взвешивание по уровням шума (SNR-based weighting / min-SNR loss weighting).
- Использовать составной лосс:
  - основной diffusion loss +
  - небольшой detail-aware компонент в latent-пространстве (например gradient/high-frequency consistency).

Это часто сильнее влияет на "мелкие детали", чем простое увеличение числа блоков.

### 3.4. Оптимизация процесса обучения (высокий приоритет)

- **EMA весов** (если еще не включен) с аккуратным decay.
- **Cosine schedule + warmup** (или one-cycle) вместо фиксированного LR.
- **Gradient clipping** для стабилизации глубокой модели.
- Проверить эффективный batch size (accumulation) и масштаб LR по batch.
- Mixed precision + `bf16` (если железо позволяет) для более стабильного долгого тренинга.

### 3.5. Архитектурные апгрейды backbone (средний/высокий приоритет)

- В каждом блоке добавить **learnable residual scaling** (`alpha`-scale) поверх attention/MLP.
- Перейти на более эффективные attention-ядра (`scaled_dot_product_attention` / Flash Attention), чтобы безопасно увеличивать ширину/глубину.
- Добавить **локальное окно + редкие глобальные блоки** (hybrid attention), чтобы сохранить детали и глобальную согласованность одновременно.

### 3.6. Data и curriculum (высокий приоритет для качества)

- Curriculum по timesteps: сначала проще/средние уровни шума, затем расширение диапазона.
- Балансировка данных по сложности сцен/структур, чтобы модель не "застревала" на средних паттернах.
- Проверка качества latent target: нет ли bottleneck в VAE/encoder-части (если латенты приходят извне).

### 3.7. Инференс и оценка (средний приоритет)

- Проверить solver/sampler (DDIM/DPM-Solver и число шагов) — качество на инференсе может ограничиваться sampler-режимом.
- Добавить валидационный набор с акцентом на fine-detail метрики (не только глобальные).
- Вести трекинг отдельно для coarse- и fine-компонент, чтобы видеть реальный эффект изменений.

## Практический roadmap (коротко)

1. Сначала рефактор в модульную схему: `InputAdapter` / `TimeConditioner` / `Backbone` / `OutputHead`.
2. Затем включить training-пакет: EMA + min-SNR weighting + scheduler warmup/cosine + grad clipping.
3. После этого сделать 2-3 целевых ablation:
   - baseline v2,
   - v2 + improved objective,
   - v2 + objective + hybrid attention.
4. Выбрать конфигурацию по метрикам и визуальной детализации на одинаковом inference-протоколе.
